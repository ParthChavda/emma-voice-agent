# Testing a live call end-to-end (ngrok + Twilio + Zoiper)

This walks through wiring up a real phone call to EMMA without needing a SIM
card: Twilio routes the call, ngrok tunnels Twilio's webhook to your laptop,
and Zoiper (a free SIP softphone) places the call over the internet.

## Prerequisites

- A Twilio account with a purchased phone number (Twilio trial accounts include one).
- `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, and `HUMAN_HANDOFF_NUMBER` set in `.env`
  (find the SID/token on the Twilio Console dashboard; `HUMAN_HANDOFF_NUMBER` is
  the E.164 number `escalate_human` calls get transferred to).
- Postgres and Qdrant running, with seed data loaded:
  ```bash
  source venv/bin/activate
  python scripts/seed_qdrant_schedules.py
  ```
- `ngrok` installed (`brew install ngrok` on macOS) and authenticated
  (`ngrok config add-authtoken <token>` — free account is enough).

## Step 1: Start the FastAPI server

```bash
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Step 2: Start ngrok

In a second terminal:

```bash
ngrok http 8000
```

Copy the `https://<random-id>.ngrok-free.app` URL ngrok prints — that's your
public address for this session (it changes every time you restart ngrok on
a free plan).

## Step 3: Point the Twilio number at your webhook

1. Twilio Console → **Phone Numbers → Manage → Active Numbers** → click your number.
2. Under **Voice Configuration**, set "A call comes in" to **Webhook**.
3. URL: `https://<your-ngrok-id>.ngrok-free.app/voice/incoming`, method **HTTP POST**.
4. Save.

This is the only Twilio-side config needed — `POST /voice/incoming` returns
TwiML that tells Twilio to open a bidirectional Media Stream back to
`wss://<your-ngrok-id>.ngrok-free.app/voice/stream`, which is where the real
STT → LLM → TTS loop runs (see `app/core/call_handler.py`).

## Step 4: Make a test call with Zoiper (no SIM needed)

Zoiper is just a SIP client — you still need *some* account that can dial a
real phone number over the internet (a free-trial SIP/VoIP provider works
fine; you're not using cellular service at any point).

1. Install [Zoiper](https://www.zoiper.com/) (desktop or mobile).
2. Add a SIP account from any VoIP/SIP trunk provider that permits outbound
   PSTN dialing (several offer free trial credit — enough for a short test call).
3. Once registered (Zoiper shows "Ready"), dial your Twilio number in full
   E.164 format, e.g. `+441612345678`.
4. Twilio answers, hits your `/voice/incoming` webhook, and connects the
   Media Stream — you should hear EMMA's greeting shortly after connecting.

## Step 5: Verify it actually worked

While the call is live, watch the `uvicorn` terminal — you'll see STT
transcripts and LLM replies logged as the conversation progresses.

After hanging up, confirm the call was recorded end-to-end:

```bash
psql "$(python3 -c 'from app.config import settings; print(settings.postgres_dsn)')" \
  -c "SELECT call_sid, patient_name, intent, escalation_flag, call_duration FROM call_summaries ORDER BY created_at DESC LIMIT 1;"
```

You should see one row for the call you just made, with a structured summary
generated by `app/core/notes.py`.

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| Twilio webhook returns an error / call fails immediately | ngrok URL changed since you last saved it in the Twilio Console — free ngrok URLs are not stable across restarts. Re-copy and re-save. |
| Call connects but EMMA never speaks | Check the `uvicorn` log for exceptions from `synthesize_speech` — usually a bad/missing `ELEVENLABS_API_KEY`. |
| EMMA doesn't understand anything said | Check `ELEVENLABS_API_KEY` and that the `uvicorn` log shows `[STT]`-style transcripts at all — if there's silence, the Media Stream may not be reaching `/voice/stream` (check the ngrok inspector at `http://127.0.0.1:4040`). |
| `escalate_human` doesn't transfer | `TWILIO_ACCOUNT_SID`/`TWILIO_AUTH_TOKEN`/`HUMAN_HANDOFF_NUMBER` missing or wrong in `.env`. |
| Nothing shows up in `call_summaries` | Check the `uvicorn` log for `[call_handler] failed to generate/save call summary: ...` — this is caught and logged, not raised, so the call itself still completes normally even if note-generation fails. |
